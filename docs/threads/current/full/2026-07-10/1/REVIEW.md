## Findings

### HIGH

**H1 — Command payloads enter the queue by live reference (no clone / no JSON-validation at `submit()`).** `world-commands.ts:71` → `command-queue.ts:21-25`; drain at `world-commands.ts:119-123`. Unlike `EventBus.emit` (which does `assertJsonCompatible` + `deepCloneJson` for the buffer AND per-listener — `event-bus.ts:19,24,28`), `submitWithResult` stores and hands out `data` by reference. Runtime-demonstrated (rev-core): reusing one command object across a submit loop makes every handler observe the last value (`[2,2,2]`, not `[0,1,2]`) — a natural pattern for an LLM emitting commands in a loop. Worse for replay: the recorder clones per-submit (records `[0,1,2]`) while the live handler applied the mutated value → `selfCheck` divergence on the engine's headline determinism guarantee. Non-JSON command data is also silently accepted when no recorder is attached. **Disposition: FIX** — clone + `assertJsonCompatible` command `data` at the submit boundary, mirroring `emit`; failing-first regression test that reuses one object across submits and asserts distinct handler-observed values. Kept HIGH (not down-weighted): it silently breaks the replay guarantee under a reasonable pattern and the sibling primitive is protected, so the asymmetry actively misleads. *(rev-core; driver-verified.)*

### Owner ruling required (HIGH-or-MEDIUM by intent) — corroborated by 2 reviewers

**H2 — The strict verified-evidence honesty gate is bypassable via `fixed`/`regressed`.** `improvement-loop.ts:213`. The evidence requirement (a replayable evidence ref + `verificationMethod`) fires ONLY for `verificationStatus === 'verified'`. `fixed`/`regressed` are strictly *stronger*, terminal "proven" claims (the fleet's "a pass is done at fixed-proven, never proposal-only") yet require no evidence and no method: `visualPlaytestFindingToImprovementFinding(v, { id, verificationStatus:'fixed' })` with no evidence builds, passes `assertImprovementFinding` (line 280), and `improvementFindingToMarker` persists it to the durable ledger — the exact "assert success without evidence" outcome the v2.0.0 strict default was built to prevent, reached by choosing `fixed` over `verified`. This faithfully implements the DOCUMENTED contract (evidence scoped to `verified`; changelog.md/api-reference), so it is a contract-**design** gap, not code-vs-contract. **Disposition: OWNER RULING** — (a) extend the gate so `fixed`/`regressed` require a replayable ref + `verificationMethod` (machine-enforce "fixed-proven" at the engine boundary; a validation-tightening behavior change with fleet + versioning impact), OR (b) explicitly document that the engine validates only `verified` and fix-proof is enforced solely by loop-ops orchestration (rerun + gates). HIGH if machine-enforcement is intended; MEDIUM as a documented scope boundary. *(rev-loop + CLI newest-diff, independently.)*

### MEDIUM

**M1 — `scenarioResultToBundle` can emit a range-gapped bundle that advertises full replayability; `openAt` then silently replays wrong state.** `session-scenario-bundle.ts:61-99`, `history-recorder.ts:160-161/361-365`, `session-replayer.ts:199-253`. `WorldHistoryRecorder` is a bounded rolling buffer (default 64 ticks / 256 commands); `runScenario` passes capacity straight through (unset → 64). The adapter sets metadata to the full `[startTick,endTick]` and copies the possibly-truncated `ticks`/`commands` with no coverage check. `openAt` guards startTick / upper bound / `replay_across_failure` / fully-empty commands, but NOT *partial* truncation → for a middle tick it hydrates from `initialSnapshot@startTick` and replays over evicted (empty) commands → silently wrong state. `snapshotAtTick` already has the `missing_tick_entries` continuity guard `openAt` lacks (`session-bundle-diff.ts:406-416`). **Disposition: FIX** — mirror that continuity guard into `openAt` (protects openAt / BundleViewer / selfCheck / fork against tick-truncation + tampering), plus make the adapter honest about command-only truncation. *(rev-replay; driver-verified.)*

**M2 — `runAgentPlaytest.ok` semantics are undocumented and inverted vs the documented `runSynthPlaytest` sibling on poisoned worlds.** `ai-playtester.ts:296-300` (`ok:false` on `poisoned`/`agentError`) vs `synthetic-playtest.ts:290` (`ok:true` on `poisoned`). api-reference lists `ok:boolean` with no prose; the code comment "Tighten ok per runSynthPlaytest" (ai-playtester.ts:293) is misleading — it is stricter, not matching. A loop driver checking `result.ok` behaves oppositely for the two runners. **Disposition: FIX** — document the deliberate divergence in api-reference + correct the comment (reconciling behavior is riskier and test-pinned). *(rev-loop; driver-verified.)*

### LOW

- **L1 — No reentrancy guard on `step()`/`stepWithResult()`.** `world-tick.ts:33,42,69`. `_inTickPhase` is tracked (`:107`) and surfaced (`isInTick`) but not checked on step entry; a system/handler/listener that calls `world.step()` re-enters `runTick()` → clobbered dirty tracking, double tick-advance, nulled metrics → wrong `TickDiff`, no error (inconsistent with the engine's fail-fast philosophy). **FIX** — throw coded `tick_reentrancy` when `_inTickPhase` is already true on entry. *(rev-core.)*
- **L2 — Recorder mutex-slot leak if `connect()` throws before its try block.** `session-recorder.ts:143` sets `world.__payloadCapturingRecorder` but the try starts at `:160`; a throw in `:144-159` (e.g. `getRegistrationManifest()` at `:156`) leaves `_connected=false`, and `disconnect()` early-returns (`:198-201`) without the slot release (`:226-228`) → the world's single-recorder slot is orphaned. Narrow trigger; defense-in-depth on the exact mutex invariant. **FIX** — release the slot on connect failure / in disconnect's early-return. *(rev-replay; driver-verified.)*
- **L3 — `no_replay_payloads` error names a remedy that also fails.** `session-replayer.ts:223` tells the user to "pass a tick that has a recorded snapshot," but `openAt(tick>startTick)` rejects ALL ticks when commands are empty. **FIX** — point to `snapshotAtTick`/`source.readSnapshot` instead. *(rev-replay.)*
- **L4 — `selfCheck` executions comparison is O(T·E) per segment.** `session-replayer.ts:412-413` re-filters an accumulating array every tick, inconsistent with the O(1) `_commandsByTick`/`_eventsByTick` Map indices the same class added. **FIX** — pre-bucket executions into a `Map<tick,…>`. *(rev-replay.)*
- **L5 — `getByTag()` iteration order is not serialize/fork round-trip-stable.** `world-tags-meta.ts:43-46` returns tag-add (insertion) order live, but after `deserialize`/`applySnapshot`/`forkAt` the index is rebuilt in ascending-id order (confirmed: `[3,1,4,0,2]` live → `[0,1,2,3,4]` post-round-trip). A consumer taking `[...getByTag('unit')][0]` gets a different entity live vs forked → replay/lockstep divergence. The last instance of the spatial-grid H1 id-sort pattern (full-review 2026-06-13). **FIX** — id-sort the returned set. *(CLI; driver-verified.)*
- **L6 — `selfCheck` no-payload short-circuit uses raw `endTick` instead of `replayableUpperBound`.** `session-replayer.ts:286`, inconsistent with every other replay bound (`openAt:201`, `tickEntriesBetween:264`, `bundle-viewer.ts:169`, `bundle-corpus.ts:208`). A legacy pre-1.1.4 bundle (`endTick===startTick`, `persistedEndTick>startTick`, commands stripped) yields spurious `stateDivergences`. **FIX** — use `replayableUpperBound(md) > md.startTick`. *(CLI.)*
- **L7 — `compareMetricsResults` emits `pctChange: ±Infinity`, which JSON-serializes to `null` and collides with the "not computable" sentinel.** `behavioral-metrics.ts:324-326`; `NumericDelta.pctChange` is `number|null`, `null` already means not-computable. In this JSON-native package a genuine 0→N growth becomes indistinguishable from a no-baseline case after round-trip. **FIX** — JSON-safe representation of unbounded growth. *(rev-loop.)*
- **L8 — `VisibilityMap.getState()` canonical ordering uses locale/ICU-dependent `localeCompare`.** `visibility-map.ts:277` (used by `getState:164,169`). For non-ASCII ids the serialized array order can differ across V8/ICU versions, so a cross-runtime `stateDigest` of visibility state could diverge. `compareCodeUnit` (`bundle-corpus-types.ts:132`) is the deterministic drop-in. Rated LOW by rev-pkg / NIT by CLI (VisibilityMap is a standalone utility, not part of World's canonical snapshot). **FIX** (cheap determinism hardening) — switch to `compareCodeUnit`. *(rev-pkg; driver-verified.)*
- **L9 — `PathCache`/`PathRequestQueue` (and `queryCache`) have no eviction → unbounded growth.** `path-service.ts:54-80`; `createGridPathCacheKey` includes the mutable `start` position, so per-tick re-pathing mints a new key every tick and old keys never evict. Memory (not correctness); standalone consumer-managed utility with `clearCache()` escape hatch. **DOCUMENT** the position-keyed-usage requirement (optional bound deferred). *(CLI + rev-core NIT-5.)*
- **L10 — `applySnapshot` does not adopt the snapshot's `strict` flag while `deserialize` does.** `world.ts:199-203` vs the in-place `applySnapshot` (`strict` is `readonly`). Defensible (strict is harness policy; replayed state is identical), but the asymmetry is undocumented. **DOCUMENT** in the api-reference / doc-comment. *(rev-core.)*
