# Detailed Devlog: 2026-04-29

## 2026-04-29 - Spec 6: Strict-Mode Determinism Enforcement (v0.8.8)

**Action:** Implemented Spec 6 (Strict-Mode Determinism Enforcement) of the AI-first dev roadmap. Adds an opt-in `WorldConfig.strict?: boolean` flag (default false). When `strict === true`, 22 mutation methods on `World` (`createEntity`, `destroyEntity`, all `*Component` methods, `setPosition`, all resource methods, `addTransfer`/`removeTransfer`, `setState`/`deleteState`, `addTag`/`removeTag`, `setMeta`/`deleteMeta`, `emit`, `random`) throw `StrictModeViolationError` when called outside a system phase / setup window / `runMaintenance(fn)` callback. Adds `World.endSetup`, `World.runMaintenance`, `World.isStrict`, `World.isInTick`, `World.isInSetup`, `World.isInMaintenance`. New module `src/world-strict-mode.ts` extracts the error class + `assertWritable` helper to keep `world.ts` from compounding existing 2379-LOC overage. ADRs 36-40 cover opt-in default-off, applySnapshot forward-compat, registration-not-gated, depth-counted reentrant maintenance, and the no-auto-maintenance-from-commit rule.

**Reasoning:** Spec 6 was the last engine-internal Tier-3 leverage point in the AI-first roadmap. With Spec 1 (recording) shipping the determinism contract as a documented invariant verified post-hoc by `selfCheck`, Spec 6 makes the contract **structural**: violations throw at the source rather than surfacing as replay divergences hours later in CI. Opt-in default-off (ADR 36) means existing consumers are unaffected; consumers ready to enforce flip the flag and audit their call sites. The depth-counted reentrant maintenance (ADR 39) and explicit-not-implicit transaction commit (ADR 40) keep the gate honest — no hidden public bypass via the transaction surface.
