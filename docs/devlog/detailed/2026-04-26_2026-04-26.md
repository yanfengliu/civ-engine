Three-reviewer consensus on the typed-generics regression (Codex M, Opus H, Gemini H) — strongest signal in the review. Two-reviewer consensus on the Layer sparsity hole (Codex H, Gemini H).

**Batch 1 fixes shipped (v0.6.0, breaking — `CommandTransaction` correctness + ergonomics overhaul):**

- **C1 — Mutable preconditions (Codex Critical):** `TransactionPrecondition` now receives a `ReadOnlyTransactionWorld<TEventMap, TCommandMap, TComponents, TState>` façade instead of the live `World`. Implemented as `Omit<World, ...write methods>` at the type level + a runtime `Proxy` in `command-transaction.ts` that throws on forbidden method names + property writes + property deletes. A side-effecting predicate now fails to typecheck and (if the type is cast away) throws at runtime: `CommandTransaction precondition cannot call '<method>': preconditions must be side-effect free`. Closes the gap between the documented "world untouched on precondition failure" guarantee and the prior implementation, which only enforced "no buffered mutation applied".
- **H1 — Typed-generics regression (3-reviewer consensus):** `CommandTransaction` is now generic over `<TEventMap, TCommandMap, TComponents, TState>`. `world.transaction()` returns the fully-threaded type. Builder methods (`setComponent`, `addComponent`, `patchComponent`, `removeComponent`) gained typed/loose overload pairs matching `World`'s pattern. `world.transaction().setComponent(e, 'hp', { wrong: 5 })` is now a TypeScript error against `World<..., ..., { hp: { current: number } }, ...>`.
- **H3 — `transaction()` skipped warnIfPoisoned (Opus High):** `World.warnIfPoisoned(api)` is now public; `CommandTransaction.commit()` calls `this.world.warnIfPoisoned('transaction')` so a poisoned world emits the standard one-time warning before applying any buffered mutation. Verified by a regression test (poisons a world, calls `transaction().commit()` twice, asserts exactly one warn emitted with the `transaction` API tag).
- **M1 — Mid-emit JSON-compat partial-apply (Opus Medium):** `assertJsonCompatible` moved from `EventBus.emit` (commit-time, after all mutations applied) to `CommandTransaction.emit()` (buffer time, before any mutation runs). A typo'd event payload now fails immediately at the builder call, leaving the world untouched. The doc previously listed this as a "v1 limitation"; the fix obsoletes that bullet.
- **L2 — Terminal-state error message confusion (Opus + Gemini Low):** introduced a private `terminalReason: 'committed' | 'aborted' | null` field, separate from the `status` machine. After `abort()`, builder methods now throw `CommandTransaction already aborted` (not `already committed`). After clean `commit()`, they throw `already committed`. After mid-throw on `commit()`, the finalize block leaves `terminalReason='committed'` (since we entered `commit` cleanly).
- **L6 — `as unknown as` cast + eslint-disables (Opus Low, falls out of H1):** `world.transaction()` no longer casts `this` to `World<..., any, any, any>`. The two `// eslint-disable-next-line @typescript-eslint/no-explicit-any` lines and the `as unknown as` cast were removed.

**Reasoning:** Picked C1+H1+H3+M1+L2+L6 as a single coherent commit because all six findings touch the same two files (`command-transaction.ts` and `world.ts:transaction()`), share the same test surface, and the type-level changes are intertwined (the read-only façade type signature wants the four generics, the typed builder overloads wants the same four generics). Splitting them across commits would have meant either landing partial type-discipline or carrying intermediate `any` bridges between commits. AGENTS.md "one version bump per coherent shipped change" — this is one change, one bump.

Per AGENTS.md "tests-first": added 7 new tests under three new `describe` blocks (`precondition read-only enforcement`, `poisoned-world warning`, `terminal status error messages`) before touching `src/`, plus updated the existing `emit at commit-time validates JSON-compat` test to match the new buffer-time semantics.

## [2026-04-26, UTC] Iter-1 batch 2 (Layer<T> correctness + perf, v0.6.1)

**Action:** Layer<T> overhaul — closes 2 High + 2 Medium + 1 Low from REVIEW.md. Non-breaking.

**Code review breakdown by AI provider and theme:** see batch 1 entry; same source review.

**Fixes shipped:**

- **H2 — Layer sparsity hole (Codex + Gemini High, two-reviewer consensus):** `setCell` / `setAt` / `fill` now compare incoming value against `defaultValue` and `cells.delete(idx)` on match (instead of storing a marker entry). `fill(defaultValue)` short-circuits to `cells.clear()`. The in-memory map and the `getState` output now stay in sync without a round-trip. Equality test: strict `===` for primitive `T`, `jsonFingerprint` comparison for object `T`.
- **H4 — Layer unbounded `structuredClone` (Gemini High):** constructor now caches `_isPrimitive = isImmutablePrimitive(defaultValue)` (true for `number | string | boolean | null`). Every read/write boundary checks `_isPrimitive` and skips `structuredClone` when true — primitives are immutable on the JS side, so the defensive-copy contract is structurally satisfied without the call. `Layer<number>` etc. are now zero-allocation across reads and writes. Object `T` paths are unchanged.
- **L5 — `clear`/`clearAt` (Opus Low, falls out of H2):** added explicit "drop this cell back to default" methods. Idempotent on already-default cells. Bounds-validated.
- **M4 — `Layer.fromState` double-validate (Opus Medium):** `assertJsonCompatible(value)` was being called explicitly per cell, then `jsonFingerprint(value)` was called, and `jsonFingerprint`'s implementation calls `assertJsonCompatible` itself — paying validation twice. Removed the explicit call. `fromState` now validates each value once.
- **M5 — `Layer.clone` double-clone (Gemini Medium):** `clone()` was implemented as `Layer.fromState(this.getState())` — two `structuredClone` passes per cell plus an intermediate `LayerState` allocation. Now implemented directly: instantiate a new layer with the same options, iterate `this.cells` once, single clone per entry (or no clone for primitive `T`).
- **Bonus** `forEachReadOnly(cb)` — added as the explicit zero-allocation traversal path for object `T` consumers who own the no-mutate discipline. Documented in api-reference. Existing `forEach` keeps its safe-clone semantics.

**Reasoning:** Picked H2+H4+L5+M4+M5 as a coherent Layer overhaul because all five touch the same file (`src/layer.ts`) and the same defensive-copy contract surface. Strip-at-write (H2) and primitive fast-path (H4) interact: the new `matchesDefault(value)` helper uses the cached `_isPrimitive` flag to pick `===` vs `jsonFingerprint`. Splitting them into separate commits would have meant carrying intermediate `if (_isPrimitive)` branching in two adjacent commits.

Per AGENTS.md tests-first: added 11 new tests (strip-at-write, clear/clearAt, primitive fast-path, forEachReadOnly) before touching `src/`. The pre-existing test that asserted `setCell(default)` filtering happens at `getState()` time still passes — the strip-at-write fix is internally consistent with the previous getState-side filter.

**Result:** 587 tests pass (up from 576). Typecheck, lint, build all clean.

## [2026-04-26, UTC] Iter-1 batch 3 (deserialize tick validation, v0.6.2)

**Action:** Single-finding commit. M2 from REVIEW.md.

**Code review breakdown by AI provider and theme:** Codex flagged this as Medium. Single-reviewer but verified ✅ in REVIEW.md.

**Fix shipped:**

- **M2 — `World.deserialize` doesn't validate snapshot.tick (Codex Medium):** previously, `gameLoop.setTick(snapshot.tick)` accepted any number — `NaN`, negative, fractional, `Infinity` — without validation. The resulting tick value flowed into `getObservableTick`, `TickDiff.tick`, the new interval scheduling check `(tick - 1) % system.interval !== system.intervalOffset` (where `(NaN - 1) % 5 === NaN`, so all interval-gated systems silently stop), and `tick++` (where `NaN++` stays `NaN`). `deserialize` now validates `Number.isSafeInteger(snapshot.tick) && snapshot.tick >= 0` and throws `WorldSnapshot.tick must be a non-negative safe integer (got <value>)` on rejection. Mirrors the validation style of `EntityManager.fromState`, `ResourceStore.fromState`, `Layer.fromState`.

**Reasoning:** Single-finding commit because M2 is independent of other findings in the review and touches a narrow boundary (`world.ts:deserialize` + 4 regression tests). Bundling it with batches 1/2 would have made those commits less coherent.

Per AGENTS.md tests-first: added 4 new tests (NaN tick, negative tick, fractional tick, Infinity tick) before the source change.

**Result:** 591 tests pass (up from 587). Typecheck, lint, build all clean.

**Fixes shipped:**

After `Number.MAX_SAFE_INTEGER` (~4.7 million years at 60 TPS), interval scheduling modulo math silently corrupts. Practical concern is zero, but the guard is one comparison. Now throws `RangeError`.
