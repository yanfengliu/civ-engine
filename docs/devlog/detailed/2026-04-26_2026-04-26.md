Three-reviewer consensus on the typed-generics regression (Codex M, Opus H, Gemini H) — strongest signal in the review. Two-reviewer consensus on the Layer sparsity hole (Codex H, Gemini H).

**Batch 1 fixes shipped (v0.6.0, breaking — `CommandTransaction` correctness + ergonomics overhaul):**

- **C1 — Mutable preconditions (Codex Critical):** `TransactionPrecondition` now receives a `ReadOnlyTransactionWorld<TEventMap, TCommandMap, TComponents, TState>` façade instead of the live `World`. Implemented as `Omit<World, ...write methods>` at the type level + a runtime `Proxy` in `command-transaction.ts` that throws on forbidden method names + property writes + property deletes. A side-effecting predicate now fails to typecheck and (if the type is cast away) throws at runtime: `CommandTransaction precondition cannot call '<method>': preconditions must be side-effect free`. Closes the gap between the documented "world untouched on precondition failure" guarantee and the prior implementation, which only enforced "no buffered mutation applied".
- **H1 — Typed-generics regression (3-reviewer consensus):** `CommandTransaction` is now generic over `<TEventMap, TCommandMap, TComponents, TState>`. `world.transaction()` returns the fully-threaded type. Builder methods (`setComponent`, `addComponent`, `patchComponent`, `removeComponent`) gained typed/loose overload pairs matching `World`'s pattern. `world.transaction().setComponent(e, 'hp', { wrong: 5 })` is now a TypeScript error against `World<..., ..., { hp: { current: number } }, ...>`.
- **H3 — `transaction()` skipped warnIfPoisoned (Opus High):** `World.warnIfPoisoned(api)` is now public; `CommandTransaction.commit()` calls `this.world.warnIfPoisoned('transaction')` so a poisoned world emits the standard one-time warning before applying any buffered mutation. Verified by a regression test (poisons a world, calls `transaction().commit()` twice, asserts exactly one warn emitted with the `transaction` API tag).
- **M1 — Mid-emit JSON-compat partial-apply (Opus Medium):** `assertJsonCompatible` moved from `EventBus.emit` (commit-time, after all mutations applied) to `CommandTransaction.emit()` (buffer time, before any mutation runs). A typo'd event payload now fails immediately at the builder call, leaving the world untouched. The doc previously listed this as a "v1 limitation"; the fix obsoletes that bullet.
- **L2 — Terminal-state error message confusion (Opus + Gemini Low):** introduced a private `terminalReason: 'committed' | 'aborted' | null` field, separate from the `status` machine. After `abort()`, builder methods now throw `CommandTransaction already aborted` (not `already committed`). After clean `commit()`, they throw `already committed`. After mid-throw on `commit()`, the finalize block leaves `terminalReason='committed'` (since we entered `commit` cleanly).
- **L6 — `as unknown as` cast + eslint-disables (Opus Low, falls out of H1):** `world.transaction()` no longer casts `this` to `World<..., any, any, any>`. The two `// eslint-disable-next-line @typescript-eslint/no-explicit-any` lines and the `as unknown as` cast were removed.

**Reasoning:** Picked C1+H1+H3+M1+L2+L6 as a single coherent commit because all six findings touch the same two files (`command-transaction.ts` and `world.ts:transaction()`), share the same test surface, and the type-level changes are intertwined (the read-only façade type signature wants the four generics, the typed builder overloads wants the same four generics). Splitting them across commits would have meant either landing partial type-discipline or carrying intermediate `any` bridges between commits. AGENTS.md "one version bump per coherent shipped change" — this is one change, one bump.

Per AGENTS.md "tests-first": added 7 new tests under three new `describe` blocks (`precondition read-only enforcement`, `poisoned-world warning`, `terminal status error messages`) before touching `src/`, plus updated the existing `emit at commit-time validates JSON-compat` test to match the new buffer-time semantics.
