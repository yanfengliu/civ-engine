[HIGH] `Metric` erases the corpus type, so the advertised user-defined extension path is not actually type-safe. In [src/behavioral-metrics.ts](C:\Users\38909\Documents\github\civ-engine\src\behavioral-metrics.ts) around `Metric.observe` and `runMetrics`, `observe` is hard-coded to plain `SessionBundle`, then `runMetrics` casts each typed bundle back to `SessionBundle` before dispatch. That means custom metrics cannot safely inspect typed `commands[].data` or `ticks[].events[].data`; they only get the default `Record<string, never>` bundle shape unless they add unsafe casts. The `runMetrics<TEventMap, TCommandMap, TDebug>` generics therefore do not flow into metric authoring, which is a real API hole for the main extension point. This needs a bundle-typed `Metric` surface and a regression test that reads typed event/command payloads from a custom metric.
