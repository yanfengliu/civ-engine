[medium] `docs/architecture/decisions.md:23` ADR 17 documents the stateful-class pattern incorrectly. It says a class method satisfies `Policy` via ``() => instance.decide.bind(instance)``, but that expression is a zero-arg factory returning a policy function, not a `Policy` itself. A consumer copying the ADR will hand the API the wrong shape and fail type-checking. The guidance should be `instance.decide.bind(instance)` (or an equivalent closure that directly returns `PolicyCommand[]` for a `PolicyContext`). Everything else in the T1 diff looks aligned with the scoped spec/plan and the T2/T3 deferrals.
